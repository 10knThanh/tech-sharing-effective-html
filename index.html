1 · Why

The problem with traditional diagrams

Draw.io Figma Excalidraw PowerPoint Mermaid
  • Diagram nhanh lỗi thời
    Project càng phát triển, hình càng xa thực tế — không ai dám tin sơ đồ cũ.
  • Không review được trên Pull Request
    File binary / XML — PR không hiện diff, kiến trúc đổi mà không ai duyệt.
  • Merge conflict không giải được
    Hai người cùng sửa một sơ đồ là phải chọn một bản, vứt bản kia.
  • Khó maintain · không đồng bộ với source code
    Diagram sống ở Drive/Figma, code sống ở Git — hai vòng đời tách rời nhau.

Documentation thường bị bỏ quên sau vài sprint.

2 · What

What is Effective HTML?

Một cách xây dựng diagram bằng HTML + CSS + component — thay vì kéo thả. Node là thẻ div, layout là CSS, diagram là DOM. 

<!-- diagram = code -->
<div class="flow">
  <div class="node">User</div>
  <div class="arrow"></div>
  <div class="node">Backend</div>
</div>

7 · Key takeaways

Bốn điều mang về

  • Diagram cũng nên được quản lý như source code.
  • Effective HTML giúp diagram dễ bảo trì, dễ review — và highlight được từng flow khi sơ đồ chồng chéo.
  • AI có thể tạo và cập nhật diagram trong vài giây.
  • Phù hợp với team đang dùng Git và Pull Request — chính là chúng ta.
